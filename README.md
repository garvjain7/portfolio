# Garv Jain — Engineering Portfolio

An interactive, AI-powered engineering portfolio designed as a **workspace-like system interface**, showcasing full-stack, AI, distributed systems, and agent-based applications built by Garv Jain.

This is not a static portfolio — it is a **system-driven experience** with:
- structured project knowledge base (RAG-ready)
- AI assistant (“Digital Garv”) for interactive querying
- deep engineering case studies
- experimental and production-grade systems

---

## 🧠 Overview

This portfolio is built to represent engineering work in a **systems-first format**, not as isolated projects.

Each project is documented with:
- architecture-level reasoning
- design tradeoffs
- system constraints
- real implementation decisions

An integrated AI assistant allows users to ask natural language questions about all projects and receive grounded responses from the knowledge base.

---

## 🚀 Key Features

### 🧩 Interactive Engineering Workspace
- 3D hero experience (in development)

- consistent dark system UI with single-accent design

### 🤖 AI Assistant (Digital Garv)
- RAG-powered chatbot over structured project knowledge

## 🏗️ Architecture

### Frontend
- Next.js 14 (App Router)
- TypeScript
- TailwindCSS
- Zustand (state management)
- R3F (3D workspace — in progress)

### Backend (planned / partial)
- FastAPI (Python)
- RAG pipeline with FAISS
- OpenAI / LLM integration for assistant
- PostgreSQL (structured storage)
- Redis (session + async tasks)

### AI System
- Retrieval-Augmented Generation (RAG)
- chunked markdown knowledge base
- strict grounding (no hallucination design)
- streaming responses
- role-based prompt control

---

## 🧠 Design Philosophy

This portfolio is built around a core principle:

> **AI systems should be constraint-driven, not generic.**

Key engineering principles followed:
- separation of observation, reasoning, and execution layers
- explicit architectural boundaries for AI systems
- preference for system design clarity over feature density
- grounding all AI outputs in verifiable knowledge
